# Sessions

ACP session IDs are native Amp thread IDs such as `T-...`.

`session/new` calls `amp threads new`, persists an initial manifest, and returns
select config options for `mode` and `effort`. No prompt tokens are spent while
creating the empty thread.

`session/load` reads the store manifest and transcript, verifies that the
server-side Amp thread can still be addressed, recreates the wrapper session
with a fresh isolated native home, and replays stored frames to the client.
`session/resume` performs the same continuability check without replaying
history.

`session/delete` writes the store tombstone first, closes any active local
session, then calls `amp threads delete`. A native delete failure is returned
after the local session is already hidden from list/load/resume, and cleanup is
retried by later list/load/resume/delete calls.

`session/close` stops accepting new prompts for the local wrapper session and
removes its isolated native home. It does not delete the server-side Amp
thread.
