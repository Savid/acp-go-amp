---
title: ACP Methods
description: The exact ACP method surface the adapter supports, and what unsupported calls return.
---

The adapter implements a fixed ACP method surface. Methods outside it return
method-not-found rather than partial behavior.

## Supported methods

| Method | Notes |
|---|---|
| `initialize` | Advertises `_meta.amp.rawEvent` and `_meta.amp.sessionStore`. |
| `authenticate` | Rejects caller method IDs; Amp owns auth. |
| `logout` | Present. |
| `session/new` | Mints the session id and persists the manifest; the Amp thread is created by the first prompt. |
| `session/load` | Replays stored transcript frames. |
| `session/resume` | Restores state without replay. |
| `session/list` | Lists active and mirrored sessions, paginated by an opaque `cursor` with pages of 50 and `nextCursor`. |
| `session/prompt` | Continues the thread with one streamed turn. |
| `session/cancel` | Interrupts the active prompt process. |
| `session/close` | Closes the local session. |
| `session/delete` | Tombstones locally, then deletes the Amp thread when one was created. |
| `session/set_config_option` | Updates `mode`. |

## Unsupported and rejected

- `session/set_mode` returns method-not-found and is not advertised.
- `_amp/session/fork` returns the package unsupported error — Amp has no
  documented native fork surface. Stable `session/fork` returns
  method-not-found.
- MCP accepts stdio and HTTP server definitions during session start; SSE and
  ACP definitions are rejected with invalid params naming the server. See
  [MCP](/docs/features/mcp).

No slash commands are advertised. Slash-prefixed text remains normal prompt
content.
