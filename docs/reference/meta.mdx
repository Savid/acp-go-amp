---
title: Meta
description: The _meta.amp metadata the adapter accepts on requests, emits on updates, and advertises on initialize.
---

Lifecycle requests carry Amp-owned settings under the `_meta.amp` namespace.
Unknown keys fail closed; foreign namespaces are ignored.

## Request metadata

```json
{
  "_meta": {
    "amp": {
      "options": {
        "env": {},
        "mode": "high"
      },
      "rawEvent": {
        "enabled": false
      }
    }
  }
}
```

## Accepted fields

| Field | Accepted values |
|---|---|
| `options.env` | Map of environment overrides. |
| `options.mode` | `low`, `medium`, `high`. |
| `options.model` | Present for symmetry; any value fails with the uniform unsupported error. |
| `options.outputSchema` | Present for symmetry; any presence fails with the uniform unsupported error. |
| `rawEvent.enabled` | `true` to stream `_amp/rawEvent` for the session. |

Unknown keys under `_meta.amp`, `_meta.amp.rawEvent`, or `_meta.amp.options`
fail closed. Foreign metadata namespaces are ignored.

## Emitted metadata

`session/update` notifications derived from delegated-agent frames — subagents,
the oracle, and the Task tool — carry `_meta.amp.parentToolUseId`, the id of the
`tool_use` block that spawned the delegated agent. Main-agent updates omit the
key. See [Updates](/docs/reference/updates) for the tagged update kinds.

## Advertised capabilities

`initialize` advertises `_meta.amp.rawEvent` and `_meta.amp.sessionStore`. It
omits fork and elicitation capability metadata, because the adapter supports
neither.

Amp therefore does not advertise `_meta["acp-go.dev/route"]`, stamp it on
callbacks, or validate a turn nonce. Its prompt-builder nonce parameter is
intentionally ignored; cancellation remains structurally scoped by `sessionId`.
