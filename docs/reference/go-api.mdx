Import the module and drive it with `Serve` or `NewAgent`. The authoritative,
always-current signatures live on
[pkg.go.dev](https://pkg.go.dev/github.com/savid/acp-go-amp); this page indexes
the surface rather than repeating godoc.

```go
import ampacp "github.com/savid/acp-go-amp"
```

## Entry points

| Symbol | Purpose |
|---|---|
| `ampacp.NewAgent(opts ...Option) *Agent` | Construct an agent you drive directly. |
| `ampacp.Serve(ctx, input, output, opts ...Option) error` | Serve ACP over the given streams. |

`ErrProcessTreeUnproven` is the stable `errors.Is` discriminator returned
when shutdown cannot prove that every Amp descendant exited. `Serve` returns
an `Agent.Close` error carrying this sentinel even when the stream loop would
otherwise return normally or with context cancellation. A host that receives
it must retain or quarantine any resources the native process tree may still
reach. Every native command has the same fence: Linux proves a dedicated
per-command subreaper empty (including descendants detached with `setsid(2)`)
only after `wait4(2)` reports `ECHILD`, and Windows proves its Job Object empty.
Session startup and native operations fail with this sentinel on platforms
without either containment backend. Parent-side proof waits are bounded, so
`Close` and `Serve` return the sentinel and retain reachable resources rather
than waiting forever.

## Common options

| Option | Purpose |
|---|---|
| `WithExecutablePath` | Native `amp` executable path. |
| `WithScratchDir` | Sole parent for all ephemeral on-disk scratch (per-session isolated `HOME`/XDG directories, startup probe directory). Empty falls back to the system temp directory; created `0700` when missing. |
| `WithHome` | Unsupported: Amp has no native config/auth root. Kept on the surface for symmetry; a non-empty value rejects at session start. Use `WithScratchDir`. |
| `WithDefaultModel` | Accepted for symmetry; non-empty values reject at session start. |
| `WithEnv` | Environment injected into native processes (e.g. `AMP_API_KEY`). |
| `WithSessionStore` | Durable mirror; see [Session store](/docs/features/session-store). |
| `WithSessionStoreLoadTimeout` | Bound on every store read (default `10s`); writes use a fixed `60s` bound. |
| `WithConcurrencyLimits` | Caps on concurrent native work. |
| `WithSeedFiles` | Relative-path → contents written into each session's isolated native `HOME` before `amp` launches. See [Seed files](/docs/features/models-config#seed-files). Accepted but inert for model routing — Amp is locked to its hosted backend. |
| `WithTurnTimeout` | Optional per-turn deadline (default `0`, no deadline); on expiry the turn fails with `cause: "timeout"`. |
| `WithRuntimeResourceHooks` | Host admission hooks for native roots and adapter scratch roots. |
| `WithTracerProvider`, `WithMeterProvider`, `WithTextMapPropagator`, `WithLogger` | Observability providers. |

## Amp per-session options

`AmpOptions` carries per-session Amp settings (also expressible as
`_meta.amp.options`):

| Field | Behavior |
|---|---|
| `Env` | Per-session environment overrides. |
| `Mode` | `low`, `medium`, or `high`. |
| `Model`, `OutputSchema` | Present for API symmetry; rejected when supplied. |

## Request builders

Helpers construct ACP requests for the session lifecycle, prompts, config
options, and MCP server declarations. See [ACP methods](/docs/reference/acp-methods)
for the method surface they target and [MCP](/docs/features/mcp) for a wiring
example.

| Builder | Purpose |
|---|---|
| `NewSessionRequest` / `LoadSessionRequest` / `ResumeSessionRequest` | Session lifecycle requests (plus a delete request). |
| `TextPromptRequest` / `PromptRequest` | Prompt requests from text or content blocks. |
| `CancelRequest` | Structural session cancellation request; Amp has no elicitation route envelope. |
| `SetConfigOptionRequest` / `SetModelRequest` | Config-option updates. |
| `StdioMCPServer` / `HTTPMCPServer` | Accepted MCP server declarations, attached with `WithSessionMCPServers`. |

The prompt builders take `(sessionID, turnNonce, ...)`. Amp has no elicitation
surface and does not advertise or stamp `acp-go.dev/route`, so it intentionally
ignores the nonce and `CancelRequest` remains structural.

`RuntimeResourceHooks` supplies `AcquireNativeRoot`, `ReserveScratchRoot`,
`ObserveProcess`, `ObserveProcessSnapshot`, and `ObserveStartupStage`.
Admission callbacks receive `runtime`, `session`, `prompt`, or `discovery`; nil
callbacks keep standalone use unbounded. Amp reserves its per-session settings
root before creation and acquires a native permit for every startup probe,
thread command, and prompt process before launch. Amp is lockless, so it never
reports home-lock supervisors. Process snapshots are emitted only for a proven
containment inventory; startup observations name `spawn`, `readiness`,
`configuration`, or `session` and carry the stage error.
