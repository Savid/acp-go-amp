Import the module and drive it with `Serve` or `NewAgent`. The authoritative,
always-current signatures live on
[pkg.go.dev](https://pkg.go.dev/github.com/savid/acp-go-amp); this page indexes
the surface rather than repeating godoc.

```go
import ampacp "github.com/savid/acp-go-amp"
```

## Entry points

| Symbol | Purpose |
|---|---|
| `ampacp.NewAgent(opts ...Option) *Agent` | Construct an agent you drive directly. |
| `ampacp.Serve(ctx, input, output, opts ...Option) error` | Serve ACP over the given streams. |

## Common options

| Option | Purpose |
|---|---|
| `WithExecutablePath` | Native `amp` executable path. |
| `WithHome` | Parent for isolated per-session native `HOME`/XDG directories. |
| `WithDefaultModel` | Accepted for symmetry; non-empty values reject at session start. |
| `WithEnv` | Environment injected into native processes (e.g. `AMP_API_KEY`). |
| `WithSessionStore` | Durable mirror; see [Session store](/docs/features/session-store). |
| `WithSessionStoreLoadTimeout` | Bound on store load (default `10s`). |
| `WithConcurrencyLimits` | Caps on concurrent native work. |
| `WithSeedFiles` | Relative-path → contents written into each session's isolated native `HOME` before `amp` launches. See [Seed files](/docs/features/models-config#seed-files). Accepted but inert for model routing — Amp is locked to its hosted backend. |
| `WithTurnTimeout` | Optional per-turn deadline (default `0`, no deadline); on expiry the turn fails with `cause: "timeout"`. |
| `WithTracerProvider`, `WithMeterProvider`, `WithTextMapPropagator`, `WithLogger` | Observability providers. |

## Amp per-session options

`AmpOptions` carries per-session Amp settings (also expressible as
`_meta.amp.options`):

| Field | Behavior |
|---|---|
| `Env` | Per-session environment overrides. |
| `Mode` | `smart`, `deep`, or `rush`. |
| `Effort` | `none` … `max`. |
| `Model`, `OutputSchema` | Present for API symmetry; rejected when supplied. |

## Request builders

Helpers construct ACP requests for the session lifecycle, prompts, config
options, and MCP server declarations. See [ACP methods](/docs/reference/acp-methods)
for the method surface they target and [MCP](/docs/features/mcp) for a wiring
example.

| Builder | Purpose |
|---|---|
| `NewSessionRequest` / `LoadSessionRequest` / `ResumeSessionRequest` | Session lifecycle requests (plus a delete request). |
| `TextPromptRequest` / `PromptRequest` | Prompt requests from text or content blocks. |
| `SetConfigOptionRequest` / `SetModelRequest` | Config-option updates. |
| `StdioMCPServer` / `HTTPMCPServer` | Accepted MCP server declarations, attached with `WithSessionMCPServers`. |
