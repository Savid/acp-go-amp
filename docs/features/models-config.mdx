title: Models And Config
description: Configure Amp turns with the select-only mode and effort options, and seed on-disk native state.
---

Amp exposes two ACP session config options — `mode` and `effort`. Both are
select-only, so a host presents a fixed value list rather than free text. There
is no ACP model option, because Amp is locked to its hosted backend.

## Config options

| Option | Accepted values | Sent to Amp as |
|---|---|---|
| `mode` | `smart`, `deep`, `rush` | `-m` |
| `effort` | `none`, `minimal`, `low`, `medium`, `high`, `xhigh`, `max` | `--effort` |

The adapter passes the selected values straight through; it does not maintain
an adapter-owned mode/effort matrix. Set them per session through
`_meta.amp.options` (`AmpOptions.Mode` / `AmpOptions.Effort`) or update them
with `session/set_config_option`.

## No model option

There is no ACP model config option for Amp. A non-empty `WithDefaultModel`,
`AmpOptions.Model`, or `_meta.amp.options.model` fails session start with the
uniform unsupported error naming field `model`.

## Seed Files

`WithSeedFiles(map[string]string)` maps relative paths to file contents that the
adapter writes into a session's resolved native root before the short-lived
`amp threads` process launches, so the native CLI can read them as its own
on-disk state.

- **Anchor** — the per-session isolated `HOME` that `amp` runs with. The adapter
  always creates a fresh isolated `HOME`/XDG root per session under the
  `-scratch-dir`/`WithScratchDir` parent (the system temp directory when that is
  unset), so seeds never land in the operator's real `HOME` or `~/.config/amp`.
- **Placement** — seeds are written under `$HOME/<relpath>`. The adapter's
  managed `settings.json` lives under `XDG_CONFIG_HOME` (alongside `HOME`), so
  seeds are written verbatim and can never overwrite the adapter's required
  settings.
- **Confinement** — absolute paths, `..` escapes, and empty keys fail closed at
  session start with the uniform unsupported error naming field `seedFiles[...]`.
  Parent directories are created `0o700` and files `0o600`.

<Warning>
Amp does not support bring-your-own-model or custom LLM endpoints — it is locked
to its hosted backend. Seed files cannot point Amp at a LiteLLM gateway or any
alternate provider and have no effect on model routing. `WithSeedFiles` is
accepted, but secrets still belong in `WithEnv`, never in a seeded file.
</Warning>
