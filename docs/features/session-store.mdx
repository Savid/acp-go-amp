# Session Store

`SessionStoreFormat` is `amp-thread-mirror-v1`.

The main subpath stores one manifest entry for the ACP session:

- `threadId`: the native Amp thread ID, which is also the ACP session ID
- `cwd`, `title`, selected `mode`, selected `effort`
- Unix millisecond create/update timestamps

The `transcript` subpath stores the full ordered Amp stream JSON frame mirror
for completed prompt turns. Each completed turn publishes the manifest and full
transcript together in one `Replace` generation. These are the frames used for
ACP `session/load` replay and raw-event fidelity. Auth material, settings
files, API keys, and secrets are never written to the store.

local transcript restore is not native thread resurrection. The local mirror is
the durability boundary for display replay; the ampcode.com thread is the
native continuation unit. continuation requires the live server-side Amp thread and AMP_API_KEY.

`session/load` hydrates the local mirror, verifies the thread can be addressed,
and replays transcript frames to the client. `session/resume` hydrates without
replaying history.

If the server-side thread is deleted or unavailable, session/load can replay the local transcript for display. A subsequent prompt fails with one terminal
`native_state_missing` error. The adapter does not create a replacement thread
from transcript text because that would lose native state.
