# Raw Events

Raw events are opt-in per session:

```json
{
  "_meta": {
    "amp": {
      "rawEvent": {
        "enabled": true
      }
    }
  }
}
```

When enabled, each captured live-turn Amp stream-json frame is emitted as
`_amp/rawEvent` with:

- `sessionId`
- `sequence`, which is per session, starts at `1`, and is strictly monotonic
  and contiguous (every event consumes exactly one sequence)
- `source` (`stream-json`)
- raw `event`

Raw events are live-turn only. They are never replayed from the session store on
`session/load`; authoritative load replay uses `session/update` frames.

Payloads are capped at 64 KiB (65536 bytes) measured on the full notification
payload. An event is never dropped and never fails the turn: an oversized event
still consumes its sequence and is emitted with `event` replaced by the fixed
marker

```json
{ "truncated": true, "reason": "oversize", "maxBytes": 65536, "sizeBytes": 0 }
```

where `sizeBytes` is the byte length of the original marshalled payload. An event
that cannot be marshalled is emitted with `{ "truncated": true, "reason":
"unserializable", "maxBytes": 65536 }` (no `sizeBytes`). A raw-event delivery
failure is recorded internally and never aborts the prompt turn. Raw events are
diagnostic and are not required for normal session replay.
