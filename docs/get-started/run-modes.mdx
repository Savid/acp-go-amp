---
title: Run Modes
description: Choose between the standalone stdio command and embedding the agent with Serve.
---

The adapter runs in two shapes with the same behavior: a standalone stdio
command that an ACP host launches as a subprocess, and an embedded `Serve` call
inside your own Go program.

## Standalone

An ACP host launches the command and speaks JSON-RPC over its stdio:

```sh
acp-go-amp -path amp -scratch-dir /path/to/scratch
```

`-scratch-dir` is a parent directory. The adapter creates one isolated directory
per ACP session under it, sets native `HOME` plus `XDG_CONFIG_HOME`,
`XDG_CACHE_HOME`, `XDG_DATA_HOME`, and `XDG_STATE_HOME` inside that directory,
and writes the per-session Amp settings file inside the isolated config tree.
The directory is removed on close/delete. When `-scratch-dir` is empty the system
temp directory is used. Amp has no native config/auth root, so `-home` is
unsupported: a non-empty value fails session start.

`-debug` writes adapter logs to stderr; ACP JSON-RPC stays on stdout only. Amp
`mode` is an ACP session config option, not a command flag. The
serve-mode flags are `-path`, `-scratch-dir`, `-home`, `-model`, `-seed-file`,
`-darwin-best-effort-containment`, `-debug`, and `-version` — see the
[CLI reference](/docs/reference/cli) for each. Darwin fails native launches
closed unless that best-effort flag is supplied; read its residual risks before
enabling it.

## Embedded

To drive Amp from inside your own host process, call `ampacp.Serve` with the
ACP input/output streams:

```go
err := ampacp.Serve(ctx, input, output,
	ampacp.WithExecutablePath("amp"),
	ampacp.WithScratchDir("/path/to/scratch"),
	ampacp.WithSessionStore(ampacp.NewInMemorySessionStore()),
)
```

Every command flag has an option equivalent (`-path` ↔ `WithExecutablePath`,
`-scratch-dir` ↔ `WithScratchDir`, `-home` ↔ `WithHome` (unsupported),
`-seed-file` ↔ `WithSeedFiles`, `-darwin-best-effort-containment` ↔
`WithDarwinBestEffortContainment`), plus embed-only options such as
`WithSessionStore` and the OpenTelemetry providers. See the
[Go API](/docs/reference/go-api) for the full set.
